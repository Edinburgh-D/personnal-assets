## 三、页面缺口汇总

### 3.1 完全缺失的页面（4个）

| 页面 | 功能 | 用户价值 | 技术复杂度 |
|------|------|----------|------------|
| **信息收集页** | 收集婚姻/孩子/年龄，路由对应题库 | 精准匹配年龄段，提升题目 relevance | 低（纯前端表单） |
| **心路历程页** | 答题后引导（"你现在是XX型，下一步可以..."） | 延长用户停留，引导到课程/再测 | 低（文案配置） |
| **海报分享页** | Canvas 2D 生成带二维码的结果海报 | 裂变传播，社交分享获客 | 中（Canvas 绘图 + 图片导出） |
| **历史记录页** | 查看过往测试记录和变化趋势 | 留存钩子，体现成长轨迹 | 中（需后端存储） |

### 3.2 需要改造的页面（3个）

| 页面 | 改造点 | 工作量 |
|------|--------|--------|
| **首页** | ① 文案：25题→28题，3种→16种 ② 入口：增加"按年龄段测试"选项 | 30分钟 |
| **答题页** | ① 计分：3维→4维 ② 题库：12道→28道 ③ 数据结构：type值映射 | 1-2小时 |
| **结果页** | ① 分数条：3条→4条 ② 画像匹配：取最高→混合算法 ③ 文案：3套→16套 | 2-3小时 |

### 3.3 需要填充数据的页面（1个）

| 页面 | 现状 | 需要填充 |
|------|------|----------|
| **课程推荐页** | 有页面无数据 | 与16种画像对应的课程列表 |

## 四、页面流程对比

### 现有流程
```
首页 → 开始测试 → 答题页（12题） → 结果页（3种之一） → 课程页（空）
```

### 设计方案流程
```
首页 → 信息收集页（婚姻/孩子/年龄） → 答题页（28题，按年龄段路由） → 结果页（16种画像） → 心路历程页（引导） → [海报分享页 或 课程推荐页]
                  ↓
            历史记录页（查看过往）
```

## 五、MVP 最小可上线 vs 完整版

### MVP（4个页面改造，约 4 小时）
- 首页（改造文案）
- 答题页（重写计分逻辑 + 替换题库）
- 结果页（重写画像匹配 + 4维展示）
- 课程页（填充基础课程数据）

**缺失功能**：信息收集页（所有用户答同一套28题）、海报分享（无裂变）、心路历程（答题结束即结束）

### 完整版（8个页面，约 2-3 天）
- 信息收集页（新增）
- 海报分享页（新增）
- 心路历程页（新增）
- 历史记录页（新增，需后端）

---

## 六、结论

| 指标 | 数字 |
|------|------|
| 现有页面 | 4 个 |
| 需要页面 | 8 个 |
| **完全缺失** | **4 个**（信息收集 / 心路历程 / 海报分享 / 历史记录） |
| 需要改造 | 3 个（首页 / 答题页 / 结果页） |
| 需要填充数据 | 1 个（课程页） |

**建议**：MVP 先不上信息收集页和海报分享，让全部用户答同一套28题，结果页有16种画像和4维分数条。这4个页面改造完就是一个可用的 v1.0。信息收集和海报分享作为 v1.1。
