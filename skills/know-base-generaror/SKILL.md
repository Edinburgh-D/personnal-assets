# Knowledge Base Generator（知识库生成器）

## Overview

生成完整的知识库HTML网页系统，包含：
- 主页面（核心概念、历史验证、与现实关系）
- 多个子页面（详细分析页面）
- 导航系统（左上角tooltip菜单 + 子页面侧边栏导航）
- 数据扭曲分析（官方数据 vs 真实感受）
- 修改记录（MD文档）
- 特殊分析页面（如消费悖论分析）

**Announce at start:** "我正在使用 knowledge-base-generator skill 来生成知识库网页系统。"

## Trigger Scenarios（触发场景）

当用户说以下类似话语时，触发此skill：
- "请帮我生成一个关于讲解'纳瓦尔宝典'的HTML网页"
- "请帮我生成一个关于'康波周期'的知识库"
- "请帮我创建一个'xxx'主题的HTML知识库"
- "我想了解'xxx'，请生成HTML网页"

## Generation Process（生成流程）

### Phase 1: 主题分析（理解用户需求）

**目标**：理解主题的核心概念、历史背景、与现实的关系。

**步骤**：
1. 分析主题的核心概念（是什么？）
2. 分析主题的历史背景（历史验证、关键事件）
3. 分析主题与现实的关系（对普通人、对国家、对社会的影响）
4. 分析主题的争议点（数据扭曲、不同观点）
5. 确定页面结构（主页面 + 子页面）

**输出**：页面结构规划表

```
| 页面 | 内容 | 章节 |
|------|------|------|
| 主页面 | 核心概念、历史验证、与现实关系 | 10-12个章节 |
| 子页面1 | 详细分析（如历史验证） | 5-7个章节 |
| 子页面2 | 详细分析（如对比分析） | 5-7个章节 |
| ... | ... | ... |
```

### Phase 2: 主页面创建

**目标**：创建主页面，包含核心概念、历史验证、与现实关系。

**主页面必须包含的章节**：

| 章节 | 内容 | 必要性 |
|------|------|--------|
| 核心概念 | 什么是xxx？定义、特征、原理 | 必须 |
| 历史背景 | 历史、关键事件、验证 | 必须 |
| 与普通人关系 | 对普通人有什么影响？应对策略 | 必须 |
| 与国家/社会关系 | 对国家/社会有什么影响？ | 推荐 |
| 数据扭曲分析 | 官方数据 vs 真实感受（如涉及经济话题） | 推荐 |
| 专业名词解释 | 相关专业名词（10个左右） | 推荐 |
| 延伸阅读 | 子页面链接 | 必须 |
| 导航菜单 | 左上角tooltip菜单 | 必须 |

**主页面HTML结构**：

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
    <meta charset="UTF-8">
    <title>[主题名称] - 知识库</title>
    <style>
        /* 基础样式 */
        /* 导航菜单样式（左上角tooltip） */
        /* 内容卡片样式 */
        /* 表格样式 */
        /* 响应式样式 */
    </style>
</head>
<body>
    <!-- 导航菜单（左上角tooltip） -->
    <div class="nav-menu-btn">
        <button onclick="toggleNavMenu()">📖 目录</button>
        <div class="nav-menu-tooltip" id="navMenuTooltip">
            <h3>📚 [主题名称]知识库导航</h3>
            <!-- 主页面章节链接 -->
            <!-- 子页面链接 -->
        </div>
    </div>
    
    <!-- 主内容 -->
    <div class="container">
        <header>
            <h1>[主题图标] [主题名称]</h1>
            <p>[主题简介]</p>
        </header>
        
        <!-- 各章节内容卡片 -->
        <div class="content-card" id="核心概念">
            <h2>[章节标题]</h2>
            <!-- 内容 -->
        </div>
        
        <!-- 更多章节 -->
    </div>
    
    <script>
        function toggleNavMenu() {
            var tooltip = document.getElementById('navMenuTooltip');
            tooltip.classList.toggle('show');
        }
    </script>
</body>
</html>
```

### Phase 3: 子页面创建

**目标**：创建多个子页面，详细分析特定主题。

**子页面类型**：

| 子页面类型 | 内容 | 示例 |
|------------|------|------|
| 历史验证页面 | 历史、关键事件、验证 | kangbo-cycle-history.html |
| 对比分析页面 | 不同国家/时期对比 | kangbo-cycle-comparison.html |
| 国家干预页面 | 政策干预、效果分析 | kangbo-cycle-government-intervention.html |
| 投资实操页面 | 实际应用、投资策略 | kangbo-cycle-investment.html |
| 技术详解页面 | 技术细节、未来预测 | kangbo-cycle-technology.html |
| 全球视角页面 | 全球分析、不同国家 | kangbo-cycle-global.html |
| 职业策略页面 | 对不同职业的影响 | kangbo-cycle-for-people.html |
| 预测方法页面 | 如何判断、应对策略 | kangbo-cycle-prediction.html |
| 城市分析页面 | 不同城市的影响 | kangbo-cycle-cities.html |
| 消费悖论页面 | 矛盾现象分析 | kangbo-cycle-consumption-paradox.html |

**子页面HTML结构**：

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
    <meta charset="UTF-8">
    <title>[子页面标题]</title>
    <style>
        /* 基础样式 */
        /* 侧边栏导航样式（右侧固定） */
        /* 内容卡片样式 */
        /* 表格样式 */
        /* 响应式样式 */
    </style>
</head>
<body>
    <!-- 侧边栏导航（右侧固定，直接展示） -->
    <div class="sidebar-nav">
        <h3>📚 页面目录</h3>
        <a href="#章节1" class="nav-item">[章节1标题]</a>
        <a href="#章节2" class="nav-item">[章节2标题]</a>
        <!-- 更多章节 -->
        <a href="[主页面].html#[锚点]" class="nav-item" style="background: linear-gradient(...); color: white;">← 返回主页面</a>
    </div>
    
    <!-- 主内容 -->
    <div class="container">
        <header>
            <h1>[子页面图标] [子页面标题]</h1>
            <p>[子页面简介]</p>
            <a href="[主页面].html#[锚点]" class="nav-link">← 返回[主题名称]主页面</a>
        </header>
        
        <!-- 各章节内容卡片 -->
        <div class="content-card">
            <h2 id="章节1">[章节标题]</h2>
            <!-- 内容 -->
        </div>
        
        <!-- 更多章节 -->
    </div>
</body>
</html>
```

### Phase 4: 导航系统

**目标**：创建完整的导航系统，方便用户跳转。

**导航系统类型**：

| 导航类型 | 位置 | 特点 | 适用页面 |
|----------|------|------|----------|
| Tooltip菜单 | 左上角 | 点击按钮展示，包含所有章节和子页面链接 | 主页面 |
| 侧边栏导航 | 右侧固定 | 直接展示，不需要点击，包含页面章节链接 | 子页面 |
| 锚点返回 | 子页面底部 | 返回主页面时自动跳转到对应章节 | 子页面 |

**导航菜单内容**：

```html
<!-- 主页面导航菜单 -->
<div class="nav-menu-tooltip" id="navMenuTooltip">
    <h3>📚 [主题名称]知识库导航</h3>
    
    <div class="nav-section">
        <div class="nav-section-title">主页面章节</div>
        <a href="#核心概念" class="nav-item">
            <div class="nav-item-title">📖 什么是[主题]</div>
            <div class="nav-item-desc">定义、特征、原理</div>
        </a>
        <!-- 更多主页面章节 -->
    </div>
    
    <div class="nav-section">
        <div class="nav-section-title">详细子页面</div>
        <a href="[子页面1].html#[锚点]" class="nav-item">
            <div class="nav-item-title">[子页面1图标] [子页面1标题]</div>
            <div class="nav-item-desc">[子页面1简介]</div>
        </a>
        <!-- 更多子页面 -->
    </div>
</div>
```

**子页面侧边栏导航**：

```html
<!-- 子页面侧边栏导航 -->
<div class="sidebar-nav">
    <h3>📚 页面目录</h3>
    <a href="#章节1" class="nav-item">[章节1图标] [章节1标题]</a>
    <a href="#章节2" class="nav-item">[章节2图标] [章节2标题]</a>
    <!-- 更多章节 -->
    <a href="[主页面].html#[锚点]" class="nav-item" style="background: linear-gradient(135deg, #667eea 0%, #764ba2 100%); color: white;">← 返回主页面</a>
</div>
```

### Phase 5: 数据扭曲分析（如涉及经济话题）

**目标**：分析官方数据与真实感受的差异，标注数据来源。

**数据扭曲分析章节**：

| 内容 | 说明 |
|------|------|
| 数据对比表 | 官方数据 vs 猜测/亲身感受 |
| 数据扭曲证据 | 具体证据（如裁员、降薪、房价） |
| 数据扭曲原因 | 统计方法、政策干预等 |
| 普通人判断方法 | 如何判断真实情况 |

**数据标注方式**：

| 标注 | 说明 |
|------|------|
| （官方） | 官方公布的数据 |
| （猜测） | 基于亲身感受推测的数据 |
| （亲身感受） | 用户实际观察到的现象 |
| ⚠️ 政策泡沫 | 政策刺激导致的虚假上涨 |

**数据扭曲分析HTML结构**：

```html
<div class="content-card" id="数据扭曲">
    <h2>⚠️ 数据扭曲分析：官方数据 vs 真实感受</h2>
    
    <div class="danger-box">
        <h4>🚨 重要提示</h4>
        <p><strong>本文档中的数据存在两种来源：</strong></p>
        <ul>
            <li><strong>官方数据：</strong>官方公布的数据（可能被美化）</li>
            <li><strong>猜测/亲身感受：</strong>基于普通人实际观察的数据（更真实）</li>
        </ul>
    </div>
    
    <div class="table-container">
        <table>
            <tr>
                <th>数据项</th>
                <th>官方数据</th>
                <th>猜测/亲身感受</th>
                <th>数据来源说明</th>
            </tr>
            <!-- 数据对比 -->
        </table>
    </div>
    
    <!-- 更多内容 -->
</div>
```

### Phase 8: 样式统一

**目标**：所有页面使用统一的样式。

**统一样式**：

| 样式 | 说明 |
|------|------|
| 渐变背景 | `linear-gradient(135deg, #667eea 0%, #764ba2 100%)` |
| 内容卡片 | 白色背景、圆角、阴影 |
| 表格样式 | 渐变标题、边框 |
| 导航样式 | Tooltip菜单或侧边栏导航 |
| 响应式 | `@media (max-width: 1200px)` 隐藏侧边栏 |
| 数据标注 | 红色（真实萧条）、黄色（虚假繁荣）、绿色（正面） |

## Notes（注意事项）

1. **数据标注**：所有数据必须标注来源（官方、猜测、亲身感受）
2. **导航系统**：主页面使用Tooltip菜单，子页面使用侧边栏导航
3. **锚点返回**：子页面返回主页面时添加锚点，返回后自动跳转到对应章节
4. **修改记录**：每次修改都要更新MD文档
5. **样式统一**：所有页面使用统一的渐变背景、内容卡片样式
6. **响应式设计**：侧边栏导航在屏幕宽度<1200px时隐藏
7. **特殊分析**：如涉及经济话题，添加数据扭曲分析和消费悖论分析
