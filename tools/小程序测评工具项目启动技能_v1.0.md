## Phase 1：情报搜集

### 执行Prompt（发给研究员）

```
你是一位敏锐的市场情报分析师，请从以下6个维度搜集关于「{产品概念}」的情报：

1. 【竞品调研】找5-10个同类产品，记录：核心功能、定价策略、获客渠道、用户评价中的抱怨点
2. 【场景题库】从小红书/知乎/微信群收集20个真实用户痛点场景，附原帖链接
3. 【技术栈方案】给出3套可行技术方案对比（成本/开发周期/扩展性），推荐一套MVP最优解
4. 【内容冷启动】提炼10条能直接发朋友圈的钩子文案+金句
5. 【法律合规】个人主体限制清单、必备合规文档、需规避的风险点
6. 【裂变增长】搜集3个同类产品的裂变机制案例，分析其海报设计/解锁策略/转化漏斗

输出格式：Markdown，每条情报附来源链接。
```

### 产出标准
- 竞品必须包含可直接体验的产品链接
- 技术方案必须包含目录结构和关键技术选型理由
- 法律部分必须可直接贴到小程序后台使用

## Phase 2：题库 + 画像体系（最核心）

### 2.1 题库设计方法论

**四维度出题法：**

每个选项对应一个维度，避免"正确答案"感：
- **A维度（情绪回应）** — 共情、接纳、先处理情绪
- **B维度（规则锚定）** — 建立边界、后果教育、坚持原则
- **C维度（探索追问）** — 提问引导、挖掘原因、培养思考
- **D维度（行动果断）** — 快速解决、转移注意力、实际行动

**选项设计原则：**
1. 每个选项25-35字，不能太长
2. 模棱两可感——每个选项都"对"，但有不同倾向
3. 避免明显的好坏之分，让用户"凭直觉选"
4. 场景必须具体（时间、地点、动作细节）

**Prompt模板（出题）：**
```
请设计一道{年龄段}的测评场景题：
- 场景：{具体场景描述，含时间/地点/动作细节}
- 选项A（情绪回应）：{共情接纳型反应}
- 选项B（规则锚定）：{建立边界型反应}
- 选项C（探索追问）：{提问引导型反应}
- 选项D（行动果断）：{快速解决型反应}
- 要求：每个选项25-35字，模棱两可，无标准答案感
```

### 2.2 画像体系设计

**四主画像 + 五混合画像：**

| 主画像 | 核心特征 | 代表色 |
|--------|---------|--------|
| 温室园丁（A型） | 高情绪回应，低规则锚定 | 薄荷绿 #98D8C8 |
| 灯塔守望者（B型） | 高规则锚定，低情绪回应 | 钴蓝 #3A5FCD |
| 探险队导师（C型） | 高探索追问，低行动果断 | 琥珀金 #D4A574 |
| 船长指挥官（D型） | 高行动果断，低探索追问 | 珊瑚红 #FF6B6B |

| 混合画像 | 组合 | 核心张力 |
|----------|------|---------|
| 温暖舵手 | A+B | 爱与规则如何平衡 |
| 智慧参谋 | B+C | 规则与自主的张力 |
| 速度疗愈师 | A+D | 情绪与效率的冲突 |
| 探险领航员 | C+D | 思考与行动的协调 |
| 全能均衡者 | A+B+C+D 均高 | 没有短板，但可能缺乏鲜明特色 |

**画像三层结构：**
```
1. 金句（一句话定义）
2. 人话版（白话解释这个画像是什么意思）
3. 肯定 → 洞察 → 建议（递进式解读）
   - 肯定：你的这个特质带给孩子什么好处
   - 洞察：这个特质可能的盲点是什么
   - 建议：一个明天就能用的具体小行动
```

## Phase 3：产品方案

### 核心流程设计

```
首页Landing → 年龄段选择 → 答题页（每题进度条） → 结果页 → 海报分享 → 解锁深度报告
```

### 首页要素
- 品牌名 + 英文slogan
- 社会证明（"已有XX人完成"）
- 视觉焦点（emoji或插画）
- 信任标签（X道题 / X种画像 / 心理学支撑）
- CTA按钮（"开始测评"）
- 底部提示（"90秒找到你的XX风格"）

### 结果页要素
- 画像名称（大字）
- 代表emoji
- 金句
- 菱形/雷达图（4维度可视化）
- 三层解读（肯定→洞察→建议）
- 分享按钮 + 预设分享话术
- 付费墙（深度报告解锁）

## Phase 4：技术架构

### MVP技术栈推荐

```
前端：Taro 3.x（微信小程序 + H5 双端）
      React + TypeScript
      样式：MVP阶段硬编码 / 可选 Tailwind CSS
      
后端：Next.js API Routes（部署在 Vercel/服务器）
      或：纯前端方案（数据存本地Storage，无需后端）
      
数据：前端JSON静态存储（题库、画像定义）
      用户数据：wx.getStorage / localStorage
      
海报：Canvas 2D API 前端绘制（750×1334竖版）
```

### 核心模块

1. **题库系统**：JSON格式，按年龄段分组，每题含场景+4选项+维度映射
2. **画像计算引擎**：纯前端，计数法（A/B/C/D各选了几个）→ 百分比 → 判断主画像/混合画像
3. **Canvas海报**：前端绘制，含品牌头图、用户画像名称、金句、四维度百分比、二维码
4. **分享链路**：小程序 onShareAppMessage + 海报图片保存

### 数据库Schema（如需后端）

```
users: id, openid, nickname, avatar, child_age, created_at
assessments: id, user_id, answers[], portrait_result, four_dimensions{}, created_at
shares: id, user_id, share_type, share_image_url, created_at
```

### MVP裁剪原则

**必须保留：**
- 首页 + 年龄段选择 + 答题页 + 结果页 + 海报
- 至少一个年龄段的完整题库（6道题）
- 4个主画像的完整解读

**可裁剪（MVP阶段）：**
- 支付/解锁深度报告（个人主体做不了微信支付）
- 多年龄段题库（先做1个年龄段验证）
- 每日一练/历史记录
- AI预演对话功能
- 后端服务器（纯前端方案）

## Phase 5：代码Prompt库

### 7个核心组件Prompt

每个Prompt必须包含：
1. 技术栈版本号
2. 设计规范（配色/字体/间距）
3. 组件完整代码（React/Taro）
4. 交互逻辑（状态管理/事件处理）
5. 异常处理

**组件清单：**
1. `LandingPage` — 首页
2. `AgeSelect` — 年龄段选择器
3. `QuestionCard` — 答题页（场景卡片+选项列表+进度条）
4. `ResultPage` — 结果页（画像展示+三层解读）
5. `PosterCanvas` — 海报绘制（Canvas 2D）
6. `calculator` — 画像计算引擎（纯函数）
7. `api/assessment` — 后端API路由（如需后端）

### Prompt模板示例（QuestionCard）

```
请用 Taro 3 + React + TypeScript 写一个 QuestionCard 组件。

设计规范：
- 主色：#FF8C42（暖橙）
- 背景：#F5F0EB（暖白）
- 字体：PingFang SC / system-ui
- 选项卡片：白底+2px灰边框，选中变橙底+橙边框
- 进度条：顶部，橙底，带动画

功能要求：
1. 接收题目数据：{ scene, options[{letter, text, dimension}] }
2. 显示场景卡片（场景标签+场景文本）
3. 显示4个纵向选项卡片（A/B/C/D圆形标签+文字）
4. 点击选中，高亮显示
5. 底部导航：上一题/下一题按钮
6. 未选时下一题按钮灰色不可点，选中后变橙色
7. 进度条根据当前题号动态计算

输出完整组件代码，包含TypeScript类型定义。
```

## Phase 6：设计原型

### HTML原型开发流程

1. **找对标产品** — 截图SBTI/16Personalities等成熟测评产品的答题页
2. **确定配色** — 暖橙(#FF8C42) + 暖白(#FFF8F0) + 炭灰(#333)
3. **写单页HTML** — 包含首页+答题页+结果页，用CSS控制显隐切换
4. **移动端测试** — 真机浏览器打开，检查触摸反馈/安全区/滚动
5. **迭代修复** — 不要disabled属性（不触发click），改用CSS类+JS检查

### 关键CSS规则

```css
/* 必须 */
-webkit-tap-highlight-color: transparent;
-webkit-appearance: none;
user-select: none;

/* 安全区适配 */
padding-bottom: calc(16px + env(safe-area-inset-bottom, 0px));

/* 按钮状态 */
.btn-next { background: #ccc; cursor: not-allowed; }
.btn-next.active { background: #FF8C42; cursor: pointer; }
```

## Phase 7：法律合规

### 必备文档（个人主体小程序）

1. **用户协议** — 服务条款、责任限制、知识产权
2. **隐私政策** — 收集哪些数据、如何使用、用户权利
3. **AI生成内容标识声明** — 如使用AI生成解读内容
4. **未成年人保护声明** — 如产品涉及儿童相关

### 个人主体限制

- ❌ 微信支付
- ❌ 流量主广告
- ❌ 社交类/游戏类
- ✅ 信息查询/工具类（可以上）

### 变现替代路径

1. 海报引流 → 个人微信/公众号
2. 深度报告 → 赞赏码/第三方收款
3. 社群运营 → 付费社群/课程
4. 验证成功后 → 注册个体户开通微信支付

## Phase 8：测试验证

### 模拟测试方法

**找10位不同背景的目标用户**，让他们完成测评并回答：
1. 哪个选项让你犹豫最久？为什么？
2. 结果画像符合你的自我认知吗？偏差在哪？
3. 你会分享这个海报到朋友圈吗？如果不会，卡在哪？
4. 你愿意为深度报告付多少钱？

### 常见问题修复清单

| 问题 | 修复方案 |
|------|---------|
| 选项太长 | 压缩到25-35字 |
| 年龄覆盖不均 | 按年龄段分组抽题 |
| 缺乏正向确认感 | 结果页先给肯定，再给洞察 |
| 海报分享率低 | 加入攀比/好奇/共鸣元素 |
| 画像太抽象 | 用人话版解释，加具体场景例子 |

## GitHub 协作流程

### 文件推送方式

**方式A（自动推送，推荐）：**
1. 用户生成 GitHub Personal Access Token（classic类型，勾选repo权限）
2. 设置remote URL：`git remote set-url origin https://TOKEN@github.com/用户名/仓库.git`
3. push 后重置为干净URL

**方式B（手动）：**
1. 我生成zip/tar.gz包发群里
2. 用户下载解压，手动拖入仓库
